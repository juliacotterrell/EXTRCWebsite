Extending Defeasible Reasoning
Beyond Rational Closure

University of Cape Town – Department of Computer Science

Overview

The EXTRC project explores ways to make Rational Closure (RC) — a core reasoning method in Artificial Intelligence — more efficient, interpretable, and accessible. Across three linked studies, the project developed:

  • A scalable reasoning engine optimised for performance.
  • A user interface and debugger to visualise and explain reasoning steps.
  • A natural language generator to translate symbolic logic into human-understandable sentences.

Background

Propositional Logic

Propositional logic is the foundation of reasoning in Artificial Intelligence. It represents knowledge using simple statements — or propositions — that can be true or false.

“Cars use petrol.”
“Electric cars are cars.”

Using classical logic, if both are true, we would also conclude “Electric cars use petrol.” This clearly isn’t correct. Classical logic assumes that once something is true, it always remains true — even when exceptions exist. This limitation motivates more flexible reasoning methods.

Defeasible Reasoning

Defeasible reasoning provides a way for computers to change their conclusions when exceptions arise. It mirrors how humans reason — we know that “cars typically use petrol,” but also accept that “electric cars are an exception.” This type of reasoning allows AI to handle uncertainty and contradiction much like humans do.

The KLM Framework

To formalize defeasible reasoning, Kraus, Lehmann, and Magidor developed the KLM Framework, which introduces typicality into logic — considering not just what is true, but what is typically true.

“Cars typically use petrol.”
“Electric cars typically do not use petrol.”

The KLM Framework ranks information by how typical or exceptional it is, allowing more specific rules (like those about electric cars) to take priority in conflicts.

Rational Closure

Rational Closure (RC) is a reasoning method built on the KLM Framework. It provides a structured way to decide what conclusions can safely be drawn in the presence of exceptions.

RC concludes: “Electric cars are still cars, but they don’t use petrol.”

RC helps AI reason cautiously — making logical conclusions while respecting exceptions. The EXTRC project improves its speed, clarity, and accessibility through optimisation, visualisation, and natural language tools.

Aims

The overarching aim is to make Rational Closure faster, clearer, and more human-friendly. Specifically, the combined objectives were to:

  • Optimise the RC algorithm for large and complex knowledge bases.
  • Design a user interface to visualise reasoning steps and outputs transparently.
  • Generate natural-language explanations that preserve logical correctness.

Collectively, these aims advance explainable and scalable reasoning for research, education, and intelligent systems.

Knowledge Base Generator

Lead: Jethro Dunn

Jethro’s work used Gemini, SBERT, and KenLM models to convert symbolic implications into clear English sentences.

  • Main Finding: LLM-generated sentences were on average 3.4× more understandable than manually written ones.
  • Consistency: SBERT and Gemini evaluations aligned strongly, validating the approach.
  • Limitations: No human trial (“gold standard”) conducted, but automated agreement confirmed reliability.
  • Contribution: Introduced a natural-language bridge between logical reasoning systems and human users, enhancing explainability.

Optimisation Techniques

Lead: Nevaniah Gounden

Implemented and benchmarked multiple optimisation strategies for RC, including parallelisation, sub-query memoization, and hybrid rank traversal with caching.

  • Performance Gains: Parallelisation reduced runtime by 85–89% on large query sets.
  • Memoization improved mixed workloads by 40–46%.
  • Combined setup achieved up to 99% faster performance.
  • Correctness: All optimised versions produced identical logical results to standard RC.

Impact: Demonstrated that RC can scale to large knowledge bases without losing logical soundness.

Interactive User Interface

Lead: Julia Cotterrell

Julia’s work created a full-stack web interface integrating the TweetyProject reasoning engine, a Spring Boot API, and a React front-end.

  • Correctness: Verified through testing — outputs matched RC definitions.
  • Performance: Local execution under 100 ms per query.
  • Usability: Expert feedback confirmed clarity and transparency, improving interpretability.

Contribution: The first tool to make RC reasoning both interactive and pedagogically accessible.
