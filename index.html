Overview

The EXTRC project explores ways to make Rational Closure (RC) ‚Äî a core reasoning method in Artificial Intelligence ‚Äî more efficient, interpretable, and accessible. Across three linked studies, the project developed:

Background

Propositional Logic

Propositional logic is the foundation of reasoning in Artificial Intelligence. It represents knowledge using simple statements ‚Äî or propositions ‚Äî that can be true or false.

‚ÄúCars use petrol.‚Äù
‚ÄúElectric cars are cars.‚Äù

Using classical logic, if both are true, we would also conclude ‚ÄúElectric cars use petrol.‚Äù This clearly isn‚Äôt correct. Classical logic assumes that once something is true, it always remains true ‚Äî even when exceptions exist. This limitation motivates more flexible reasoning methods.

Defeasible Reasoning

Defeasible reasoning provides a way for computers to change their conclusions when exceptions arise. It mirrors how humans reason ‚Äî we know that ‚Äúcars typically use petrol,‚Äù but also accept that ‚Äúelectric cars are an exception.‚Äù This type of reasoning allows AI to handle uncertainty and contradiction much like humans do.

The KLM Framework

To formalize defeasible reasoning, Kraus, Lehmann, and Magidor developed the KLM Framework, which introduces typicality into logic ‚Äî considering not just what is true, but what is typically true.

‚ÄúCars typically use petrol.‚Äù
‚ÄúElectric cars typically do not use petrol.‚Äù

The KLM Framework ranks information by how typical or exceptional it is, allowing more specific rules (like those about electric cars) to take priority in conflicts.

Rational Closure

Rational Closure (RC) is a reasoning method built on the KLM Framework. It provides a structured way to decide what conclusions can safely be drawn in the presence of exceptions.

RC concludes: ‚ÄúElectric cars are still cars, but they don‚Äôt use petrol.‚Äù

RC helps AI reason cautiously ‚Äî making logical conclusions while respecting exceptions. The EXTRC project improves its speed, clarity, and accessibility through optimisation, visualisation, and natural language tools.

Aims

The overarching aim is to make Rational Closure faster, clearer, and more human-friendly. Specifically, the combined objectives were to:

  • Optimise the RC algorithm for large and complex knowledge bases.
  • Design a user interface to visualise reasoning steps and outputs transparently.
  • Generate natural-language explanations that preserve logical correctness.

Collectively, these aims advance explainable and scalable reasoning for research, education, and intelligent systems.
