Optimisation Techniques for Rational Closure

Abstract

This project investigates how the computational efficiency of Rational Closure (RC) in defeasible reasoning can be improved without affecting logical soundness. RC provides a way for artificial intelligence systems to reason about exceptions and uncertainty, but its sequential nature limits scalability. Three optimisation techniques were implemented — parallelisation, sub-query memoization, and multi-rank traversal — to reduce runtime across knowledge bases of increasing size and complexity. Together, these techniques achieved up to a 95–99% reduction in runtime compared to the baseline sequential implementation :contentReference[oaicite:3]{index=3}.

Introduction & Aims

Rational Closure (RC) provides a structured approach for drawing cautious conclusions when information contains exceptions. Although theoretically robust, RC’s sequential entailment checks make it slow for large knowledge bases. This work aims to:

  • Optimise RC’s runtime performance through targeted computational enhancements.
  • Preserve the algorithm’s integrity and logical correctness during optimisation.
  • Evaluate scalability across different dataset sizes and query types.

By applying parallelisation, caching, and adaptive traversal methods, the study sought to make Rational Closure practical for large-scale reasoning in explainable AI systems :contentReference[oaicite:4]{index=4}:contentReference[oaicite:5]{index=5}.

Related Work

Prior implementations of Rational Closure relied on sequential entailment, as first formalised by Lehmann and Magidor. Although sound, these versions suffered from exponential time growth as knowledge bases increased in complexity. Later work by Casini, Giordano, and Straccia introduced syntactic and semantic improvements but retained the sequential core. Recent studies explored scalability through caching, distributed computing, and graphical reasoning tools, yet few directly addressed optimisation of RC’s core execution loop. This project builds upon these foundations by experimentally comparing multiple optimisation strategies under identical conditions to evaluate both correctness and speed.

Optimisation Techniques

1. Parallelisation

Parallelisation executes entailment checks for multiple queries simultaneously, leveraging multi-core processing to reduce wall-clock time. Each query runs as an isolated task, ensuring deterministic results while maintaining thread safety. This approach achieved average speed-ups of 85–89% across large knowledge bases, with identical outputs to the sequential baseline.

2. Sub-query Memoization

Memoization stores the results of previously computed sub-queries to prevent redundant computation. Each entailment check generates a unique key from the query and the ranked knowledge base, ensuring correctness and reusability. This reduced execution time by 40–46% for workloads containing repeated or partially overlapping queries.

3. Multi-Rank Traversal

Traditional RC checks ranks linearly. Multi-rank traversal introduces adaptive strategies that switch between Naive, Binary, or Ternary search depending on the depth of the knowledge base. For small rank depths, linear scanning is efficient; for deeper structures, ternary traversal reduces the total number of checks. This adaptability preserved correctness while improving scalability for deeper or more complex reasoning tasks.

Conclusion

The combined use of parallelisation, caching, and adaptive traversal produced substantial performance improvements without altering logical outcomes. The best-performing configurations paired parallelisation with cached traversal, achieving up to 99% faster execution relative to the baseline. These findings demonstrate that Rational Closure — a foundational logic for defeasible reasoning — can be efficiently scaled for practical AI applications without sacrificing correctness.
